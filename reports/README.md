## Report vs Repository Documentation

| Resource | Format | Audience | Contents |
|----------|--------|----------|----------|
| `final-report.pdf` | PDF | Academic / stakeholder review | Narrative write-up, likely includes introduction, methodology discussion, results interpretation, and conclusions |
| [`../docs/`](../docs/) | Markdown | Developers / reproducibility | Structured technical reference: methodology, results tables, setup guide |
| [`../README.md`](../README.md) | Markdown | New visitors | Project overview and quick start |

The PDF report and the markdown documentation complement each other: the report provides the narrative context, while `docs/` provides machine-readable reference material for rerunning and extending the pipeline.
