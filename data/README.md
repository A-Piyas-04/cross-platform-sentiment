## Raw Data

### Required Files

| File | Platform | Expected text column | Label column |
|------|----------|----------------------|--------------|
| `Reddit_Data.csv` | Reddit | `clean_comment` | `category` |
| `Twitter_Data.csv` | Twitter | `clean_text` | `category` |

### Label Schema

Both files use the same integer sentiment labels:

| Value | Sentiment |
|-------|-----------|
| `-1` | Negative |
| `0` | Neutral |
| `1` | Positive |

The notebook **keeps only labels present in both datasets** (intersection of label sets). In practice this retains all three classes above.

## Preprocessing Summary

The notebook applies the following steps before modeling:

1. **Column standardization** — rename to `text` and `emotion`
2. **Label cleaning** — drop rows with missing labels; cast to integer
3. **Label intersection** — keep only emotions shared by both platforms
4. **Text cleaning** — lowercase, remove URLs and `@mentions`, normalize hashtags, collapse whitespace
5. **Length filter** — remove texts with `len(text_clean) <= 10`
6. **Class balancing** — sample up to **1,000 rows per class per platform** (`random_state=42`)

### Processed Output Sizes

| File | Rows | Class distribution |
|------|------|--------------------|
| `reddit_processed.csv` | 3,000 | 1,000 per class (-1, 0, 1) |
| `twitter_processed.csv` | 3,000 | 1,000 per class (-1, 0, 1) |

## Processed File Schemas

### `reddit_processed.csv` / `twitter_processed.csv`

Generated after cleaning and balancing.

| Column | Type | Description |
|--------|------|-------------|
| `text` | string | Original text field (standardized column name) |
| `emotion` | int | Sentiment label: -1, 0, or 1 |
| `text_clean` | string | Cleaned text used for modeling |

### `reddit_linguistic.csv` / `twitter_linguistic.csv`

Per-sample linguistic feature tables used in compression analysis.

| Feature | Description |
|---------|-------------|
| `type_token_ratio` | Unique tokens / total tokens |
| `avg_word_length` | Mean character length of words |
| `exclamation_dens` | Exclamation marks per token |
| `question_dens` | Question marks per token |
| `emoji_count` | Emoji occurrences |
| `positive_ratio` | VADER positive score |
| `negative_ratio` | VADER negative score |
| `sentiment_polarity` | TextBlob polarity |
| `first_person_dens` | Density of first-person pronouns |
| `sentence_count` | Number of sentences |

Platform-level aggregates are saved to `results/compression_by_emotion.csv`.

---

## Data Quality Notes

- Twitter labels may arrive as floats (`-1.0`, `0.0`, `1.0`); the notebook casts them to integers.
- Text is already partially cleaned in the source files (`clean_comment`, `clean_text`); the notebook applies an additional normalization pass.
- Balancing uses random subsampling with a fixed seed for reproducibility.
